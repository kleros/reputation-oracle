# Quick Task 260423-nw1: Stop Hook for End-of-Turn Linting

## Goal

Add a Claude Code Stop hook that runs linters at end-of-turn and blocks the stop (feeds errors back to the model) when any fail. Prevents lint drift like agentkit's.

## Reference

`/Users/jaybuidl/project/kleros/agentkit/.claude/hooks/lint-check.sh` — structure + contract:
- Read stdin JSON, bail if `stop_hook_active:true` (prevents retry loop)
- Skip if the tool isn't configured (portability)
- On failure: emit to stderr + exit 2 to block the stop and feed output back

## Stack adaptation

This repo has two lintable surfaces; agentkit is single-stack (pnpm).

| Surface | Command | Dir |
|---------|---------|-----|
| Bot (TS, Biome) | `npm run lint` → `biome check .` | `bot/` |
| Contracts (Solidity, Foundry) | `forge fmt --check` | `contracts/` |

Notes:
- Root package is empty (no `package.json` with scripts). All lint runs live in subdirs.
- Use `npm` — project has `package-lock.json`, not pnpm. (Memory: project uses npm for bot.)
- `forge fmt --check` is the only idiomatic Solidity linter in use; `forge build` is not a lint step.
- Guard each surface with existence checks so the hook stays portable and silent when a surface isn't present.
- Aggregate failures — run BOTH, report BOTH, don't short-circuit. A single exit 2 blocks the stop.

## Plan

### Task 1 — Create hook script `.claude/hooks/lint-check.sh`

Adapted from agentkit reference:

```bash
#!/usr/bin/env bash
# Stop hook: run bot + contracts linters at end-of-turn. Block the stop
# (feed errors back to the model) if any fail. Silent on success.

set -u

INPUT=$(cat)

# Prevent infinite retry loop: if Claude Code already invoked us and the model
# is now trying to stop again after our feedback, let it through.
if printf '%s' "$INPUT" | grep -q '"stop_hook_active":[[:space:]]*true'; then
  exit 0
fi

cd "${CLAUDE_PROJECT_DIR:-$(pwd)}" || exit 0

FAILED=()
OUTPUTS=()

# Bot: biome via npm
if [ -f bot/package.json ] && grep -q '"lint"[[:space:]]*:' bot/package.json 2>/dev/null; then
  BOT_OUTPUT=$(cd bot && npm run --silent lint 2>&1)
  BOT_EXIT=$?
  if [ "$BOT_EXIT" -ne 0 ]; then
    FAILED+=("bot lint (exit $BOT_EXIT)")
    OUTPUTS+=("── bot ──"$'\n'"$BOT_OUTPUT")
  fi
fi

# Contracts: forge fmt --check
if [ -f contracts/foundry.toml ] && command -v forge >/dev/null 2>&1; then
  FORGE_OUTPUT=$(cd contracts && forge fmt --check 2>&1)
  FORGE_EXIT=$?
  if [ "$FORGE_EXIT" -ne 0 ]; then
    FAILED+=("forge fmt (exit $FORGE_EXIT)")
    OUTPUTS+=("── contracts ──"$'\n'"$FORGE_OUTPUT")
  fi
fi

if [ ${#FAILED[@]} -eq 0 ]; then
  exit 0
fi

{
  echo "Lint failed (${FAILED[*]}). Fix these before ending the turn:"
  echo
  for O in "${OUTPUTS[@]}"; do
    echo "$O"
    echo
  done
} >&2
exit 2
```

**Files:** `.claude/hooks/lint-check.sh`
**Action:** Create file, `chmod +x`.
**Verify:** `bash -n .claude/hooks/lint-check.sh` parses clean; manual smoke test: pipe `'{}'` → exit 0 (success case); pipe `'{"stop_hook_active":true}'` → exit 0 (loop guard).
**Done:** File exists, executable, parses, smoke-tests pass.

### Task 2 — Wire the hook into `.claude/settings.json`

Add a `Stop` section to `hooks`:

```json
"Stop": [
  {
    "hooks": [
      {
        "type": "command",
        "command": "bash \"$CLAUDE_PROJECT_DIR\"/.claude/hooks/lint-check.sh"
      }
    ]
  }
]
```

**Files:** `.claude/settings.json`
**Action:** Insert `Stop` key into `hooks` object, preserving existing `SessionStart`/`PostToolUse`/`PreToolUse` blocks.
**Verify:** `jq . .claude/settings.json >/dev/null` — valid JSON. `jq '.hooks.Stop' .claude/settings.json` returns the block.
**Done:** JSON valid; Stop hook present with the expected command string.

## must_haves

**Truths:**

- Guards against retry loop via `stop_hook_active`.
- Silent on success, loud on failure (exit 2 + stderr feeds back to the model).
- Skips surfaces that aren't present (portability — doesn't fail if `forge` missing).

**Artifacts:**
