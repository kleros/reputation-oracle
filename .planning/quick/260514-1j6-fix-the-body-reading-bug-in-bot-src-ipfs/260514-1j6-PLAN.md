for structured Pinata errors.

**bot/src/ipfs.ts — replace lines 80-90 (the `if (!response.ok)` block body):**

Replace the current try/catch that calls `response.json()` then falls back to `response.text()` with a text-first approach:

```ts
if (!response.ok) {
  const errorClass = classifyHttpStatus(response.status);
  let errorBody: string;
  const raw = await response.text().catch(() => "");
  if (raw === "") {
    errorBody = "(empty body)";
  } else {
    try {
      const json = JSON.parse(raw) as { error?: unknown };
      const e = json.error;
      if (e && typeof e === "object" && !Array.isArray(e)) {
        const nested = e as { reason?: string; details?: string };
        errorBody = [nested.reason, nested.details].filter(Boolean).join(": ").slice(0, 500) || raw.slice(0, 500);
      } else if (typeof e === "string") {
        errorBody = e.slice(0, 500);
      } else {
        errorBody = raw.slice(0, 500);
      }
    } catch {
      errorBody = raw.slice(0, 500);
    }
  }
  const err = Object.assign(new Error(`Pinata ${response.status}: ${errorBody}`), { errorClass });
  throw err;
}
```

Key properties of this fix:
- `response.text()` is called once — stream is never double-consumed.
- Empty string → `(empty body)` literal.
- Pinata nested object `{error:{reason,details}}` → joined string e.g. `"FORBIDDEN: Account blocked due to plan usage limit"`.
- Pinata flat string `{error:"some message"}` → that string directly.
- Non-JSON (HTML) → raw text.
- Any other JSON shape → raw serialized text (via `raw.slice`).
