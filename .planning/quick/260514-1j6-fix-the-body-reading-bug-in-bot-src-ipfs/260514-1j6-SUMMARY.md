# Phase quick-260514-1j6 Plan 01: Fix Pinata error-body parser Summary

**One-liner:** Text-first error-body parser eliminates consumed-stream fallback bug and `[object Object]` coercion for Pinata structured errors.

## What Was Built

Fixed `bot/src/ipfs.ts` error block (lines 80-95): replaced `response.json()` + `text()` fallback pattern with a single `response.text()` call followed by `JSON.parse`. This eliminates the production bug where `response.json()` consumed the stream and the `text()` fallback found it locked, producing `"(unreadable body)"`. Secondary fix: Pinata's `{error:{reason,details}}` object shape is now properly serialized to `"REASON: details"` instead of `[object Object]`.

Added three regression tests (Tests 11-13) covering the three fixed paths. Updated `makeErrorResponse` factory in `ipfs.test.ts` to text-only (removed `json` method that is no longer called).
