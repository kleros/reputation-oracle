Create `deploy/systemd/` directory and write the service unit template with EXACTLY the following content (copy verbatim — no additions, no omissions):

```ini
[Unit]
Description=Kleros Reputation Oracle (%i)
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
RemainAfterExit=no
User=oracle
Group=oracle
WorkingDirectory=/opt/reputation-oracle/bot
EnvironmentFile=/etc/reputation-oracle/%i.env
ExecStart=/usr/bin/node --import tsx /opt/reputation-oracle/bot/src/index.ts
StandardOutput=journal
StandardError=journal
SyslogIdentifier=reputation-oracle-%i
TimeoutStartSec=300
ProtectSystem=strict
PrivateTmp=true
NoNewPrivileges=true
```

CRITICAL CONSTRAINTS (per D-12, D-14, D-15):
- `%i` is the systemd instance specifier — DO NOT replace with "sepolia" or any literal string
- NO `Restart=` directive — omitting it defaults to `Restart=no`, which is correct for Type=oneshot + timer
- NO `ProtectHome=true`, NO `MemoryMax=`, NO `CPUQuota=`, NO `ReadWritePaths=` — exactly 4 hardening directives and no more
- ExecStart uses absolute path `/usr/bin/node` (NodeSource apt installs here) — NOT `node` (PATH not available in systemd)
- NO `--env-file=` in ExecStart — `EnvironmentFile=` directive already injects the env (redundant and confusing if both used)
- NO `[Install]` section on the service — the timer owns the WantedBy relationship; service is triggered by the timer, not enabled directly

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systemd unit file → process privileges | Unit file directives define what oracle user can and cannot do |
| EnvironmentFile path → secret values | Path must point to 0600 file; if path is wrong, service fails safely |
| journald → log retention | Caps prevent disk exhaustion; too low = evidence loss |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-02-01 | Information Disclosure | `systemctl show reputation-oracle@sepolia` exposes env vars | mitigate | `EnvironmentFile=` used exclusively for secrets (D-18); no `Environment=KEY=secret` directives in unit file; only non-secret config like LOG_LEVEL may use `Environment=`, but Phase 7 does not add any |
| T-07-02-02 | Elevation of Privilege | Service running as root if User= omitted | mitigate | `User=oracle` and `Group=oracle` explicit in [Service]; `NoNewPrivileges=true` prevents privilege escalation post-start |
| T-07-02-03 | Tampering | World-writable unit file allows service redefinition | mitigate | Files installed via `cp` by bootstrap running as root; final ownership root:root mode 0644 (systemd default for /etc/systemd/system/); `ProtectSystem=strict` prevents oracle user from modifying /etc/ |
| T-07-02-04 | Denial of Service | Missing TimeoutStartSec lets hung bot block timer forever | mitigate | `TimeoutStartSec=300` set (D-15/P1-04); systemd kills after 5 min; next timer interval fires normally |
| T-07-02-05 | Denial of Service | Restart= causes thrash spiral on systemic failure | mitigate | No `Restart=` directive (D-14/P1-03); timer is the retry mechanism; one failed run does not trigger immediate re-run |
| T-07-02-06 | Denial of Service | journald disk exhaustion causes silent log loss | mitigate | `SystemMaxUse=500M` cap via drop-in (D-25/P1-08); 500MB >> expected daily log volume |
</threat_model>
