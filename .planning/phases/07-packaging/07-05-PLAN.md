## 1. Fresh VPS Install

**Prerequisites:** Ubuntu 24.04 VPS, SSH access as root or sudo user.

**Two commands to a running-ready oracle:**

```bash
# Clone repo to standard location (root owns /opt/reputation-oracle initially)
sudo git clone https://github.com/kleros/reputation-oracle /opt/reputation-oracle

# Run bootstrap (creates oracle user, installs Node 22, sets up systemd, creates env stub)
cd /opt/reputation-oracle && sudo ./deploy/bootstrap.sh
```

Bootstrap is idempotent — safe to re-run if it fails partway through.

**What bootstrap does** (in order):
1. Installs git, curl, ca-certificates via apt
2. Installs Node 22 LTS via NodeSource apt (skips if already v22)
3. Creates `oracle` system user (no shell, no home)
4. Creates `/etc/reputation-oracle/` directory
5. Transfers repo ownership to `oracle:oracle`
6. Runs `npm ci --omit=dev` as oracle (installs production dependencies)
7. Creates `/etc/reputation-oracle/sepolia.env` stub at mode 0600 owned by oracle (skips if already exists)
8. Installs systemd unit templates to `/etc/systemd/system/`
9. Installs journald retention drop-in to `/etc/systemd/journald.conf.d/`
10. Reloads systemd daemon
11. Restarts systemd-journald to apply retention caps

**What bootstrap does NOT do:** Enable or start the timer. That is your explicit action after dry-run validation (§3 → §4).

## 2. Secret Fill Procedure

After bootstrap completes, fill the secrets stub with real values.

```bash
sudo -u oracle nano /etc/reputation-oracle/sepolia.env
```

**Required values to fill:**

| Key | Description |
|-----|-------------|
| `RPC_URL` | Sepolia RPC endpoint (e.g. Alchemy: `https://eth-sepolia.g.alchemy.com/v2/<key>`) |
| `ROUTER_ADDRESS` | Deployed KlerosReputationRouter proxy address on Sepolia |
| `BOT_PRIVATE_KEY` | 0x-prefixed private key of the authorized bot signer |
| `SUBGRAPH_URL` | Goldsky subgraph endpoint (default in stub is correct for Sepolia) |

**Leave unchanged:** `CHAIN_ID=11155111` and `PGTCR_ADDRESS` (pre-filled with correct Sepolia values).

**Phase 8 keys** (`BETTERSTACK_SOURCE_TOKEN`, `BETTERSTACK_HEARTBEAT_TOKEN`) — leave commented out until Phase 8 Observability is set up.

> **WARNING: Never use `echo`, `cat`, or shell redirection to populate secret values.**
> Bash history captures every command — a `echo "BOT_PRIVATE_KEY=0x..."` entry in `~/.bash_history`
> is a permanent plaintext record of your private key. Always edit directly via `nano` or `vim`.

Verify permissions after editing:

```bash
stat -c '%a %U %G' /etc/reputation-oracle/sepolia.env
# Expected output: 600 oracle oracle
```

## 3. First-Run Verification (Dry Run)

After filling secrets, run the bot in dry-run mode as the `oracle` user to confirm everything works before enabling the live timer.

```bash
sudo -u oracle --preserve-env=HOME \
  env $(cat /etc/reputation-oracle/sepolia.env | grep -v '^#' | grep -v '^$' | xargs) \
  /usr/bin/node --import tsx /opt/reputation-oracle/bot/src/index.ts --dry-run
```

**Acceptance criteria (all four must hold):**

| Check | How to verify |
|-------|---------------|
| Exit code 0 | `echo $?` after the command |
| `RunSummary` in stdout | Look for a JSON line with `"type":"RunSummary"` |
| `itemsFetched > 0` | Check `itemsFetched` field in the RunSummary JSON |
| `chainId == 11155111` | Check `chainId` field in the RunSummary JSON |
| No `level:error` or `level:fatal` in stderr | Inspect the stderr NDJSON lines |
| No files written to disk | Bot is stateless; `ls /tmp | wc -l` should not increase |

**Parsing the RunSummary:**

```bash
# Capture stdout and pretty-print the RunSummary
sudo -u oracle --preserve-env=HOME \
  env $(cat /etc/reputation-oracle/sepolia.env | grep -v '^#' | grep -v '^$' | xargs) \
  /usr/bin/node --import tsx /opt/reputation-oracle/bot/src/index.ts --dry-run \
  | grep '"type":"RunSummary"' | python3 -m json.tool
```

**If the dry run fails:** See §8 Troubleshooting for common causes (tsx missing, wrong permissions, Node path, etc.).

## 4. Enabling the Timer

Once the dry run passes, enable the Sepolia timer:

```bash
sudo ./deploy/start-timer.sh sepolia
```

This calls `systemctl enable --now reputation-oracle@sepolia.timer`, which:
- Enables the timer to survive reboots
- Starts the first 15-minute countdown immediately

**Verify the timer is active:**

```bash
systemctl status reputation-oracle@sepolia.timer
systemctl list-timers reputation-oracle@sepolia.timer
```

**Verify the first run completes:**

```bash
journalctl -u reputation-oracle@sepolia -f
# Wait for the first run to appear (fires ~2min after boot, then every 15min)
```

## 5. Update Flow

To deploy a new version:

```bash
sudo /opt/reputation-oracle/deploy/update.sh sepolia
```

This script atomically: stops the timer → `git pull --ff-only` → `npm ci --omit=dev` → starts the timer.

At most one scheduled run is skipped (< 15 min gap). The stateless bot catches up fully on the next run.

**If update.sh fails mid-sequence:** Re-run the same command. The script is idempotent. If only the timer needs to be restored:

```bash
sudo systemctl start reputation-oracle@sepolia.timer
```

---

## 6. Rollback

If a new version causes failures, roll back to the previous commit:

```bash
# Stop the timer first
sudo systemctl stop reputation-oracle@sepolia.timer

# Find the previous working commit
sudo -u oracle git -C /opt/reputation-oracle log --oneline -10

# Reset to it
sudo -u oracle git -C /opt/reputation-oracle reset --hard <commit-hash>

# Reinstall deps for that version
sudo -u oracle npm --prefix /opt/reputation-oracle/bot ci --omit=dev

# Restart timer
sudo systemctl start reputation-oracle@sepolia.timer
```

Verify the rollback with a dry run (§3) before relying on it.

## 7. Time Sync Check

The bot uses `Date.now()` for evidence timestamps. If VPS clock drifts, evidence timestamps will be off.

```bash
timedatectl status
```

Expected output should show:
- `System clock synchronized: yes`
- `NTP service: active`

If NTP is not active:

```bash
sudo systemctl enable --now systemd-timesyncd
```

## 8. Troubleshooting

### Common Failures

| Symptom | Likely cause | Diagnosis | Fix |
|---------|-------------|-----------|-----|
| `Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'tsx'` | `tsx` in devDependencies instead of dependencies (P1-02) | `node -e "const p=JSON.parse(require('fs').readFileSync('/opt/reputation-oracle/bot/package.json','utf8')); console.log('tsx in deps:', 'tsx' in p.dependencies);"` | Run `npm install tsx` in bot/ then redeploy via `update.sh` |
| `stat /etc/reputation-oracle/sepolia.env → 644` or world-readable | Wrong permissions (P1-05) | `stat -c '%a %U %G' /etc/reputation-oracle/sepolia.env` | `chmod 0600 /etc/reputation-oracle/sepolia.env && chown oracle:oracle /etc/reputation-oracle/sepolia.env` |
| Service fails: `No such file or directory: /home/oracle/.nvm/...` | nvm Node used instead of system Node (P1-07) | `systemctl show reputation-oracle@sepolia.service | grep ExecStart` | Reinstall Node via NodeSource apt; ExecStart must use `/usr/bin/node` |
| `journalctl -u reputation-oracle@sepolia` shows empty output or gaps | journald retention cap not applied (P1-08) | `journalctl --disk-usage` | Run bootstrap again (Step 9-11) or: `systemctl restart systemd-journald` |
| Bot exits with `level:fatal` config error | Missing or malformed env var | `journalctl -u reputation-oracle@sepolia -n 30 -o cat` | Check `/etc/reputation-oracle/sepolia.env` — all required keys must be filled without quotes |
| Timer shows `inactive (dead)` instead of `active (waiting)` | Timer never enabled | `systemctl is-enabled reputation-oracle@sepolia.timer` | Run `sudo ./deploy/start-timer.sh sepolia` |
| `permission denied` running bootstrap.sh | Not run as root | `whoami` | `sudo ./deploy/bootstrap.sh` |

### Useful Commands

```bash
# View recent runs
journalctl -u reputation-oracle@sepolia -n 50

# Follow live output
journalctl -u reputation-oracle@sepolia -f

# Check timer next fire time
systemctl list-timers reputation-oracle@sepolia.timer

# Check service last exit code
systemctl show reputation-oracle@sepolia.service | grep ExecMainStatus

# Check journald disk usage
journalctl --disk-usage

# Verify oracle user exists
id oracle
