<feature>
  <name>computeActions — stateless diff engine</name>
  <files>bot/src/diff.ts, bot/test/diff.test.ts</files>
  <behavior>
    Scenario 1 — Positive feedback (Submitted/Reincluded + None or Negative):
    - Submitted + None -> submitPositiveFeedback
    - Reincluded + None -> submitPositiveFeedback
    - Submitted + Negative (re-registration after dispute) -> submitPositiveFeedback
    - Submitted + Positive -> NO ACTION (already has positive)

    Scenario 2 — Negative feedback (Absent + Reject + Positive):
    - Absent + disputeOutcome="Reject" + Positive -> submitNegativeFeedback
    - Absent + disputeOutcome="Reject" + None -> NO ACTION (nothing to negate)
    - Absent + disputeOutcome="Reject" + Negative -> NO ACTION (already negative)

    Scenario 3 — Revoke only (Absent + voluntary withdrawal + Positive):
    - Absent + disputeOutcome=null + Positive -> revokeOnly
    - Absent + disputeOutcome="None" + Positive -> revokeOnly
    - Absent + disputeOutcome="Accept" + Positive -> revokeOnly
    - Absent + no dispute + None -> NO ACTION (nothing to revoke)

    Edge cases:
    - Empty items array -> empty actions array
    - Multiple items -> produces correct action per item
    - Unknown agentId (not in routerStates map) -> treated as FeedbackType.None
  </behavior>
  <implementation>
    Function signature per D-04:
    ```typescript
    export function computeActions(
      items: ValidatedItem[],
      routerStates: Map<bigint, FeedbackType>
    ): Action[]
    ```

    Logic per D-06:
    - For each item, get currentType from routerStates (default: FeedbackType.None)
    - Scenario 1: (status === "Submitted" || status === "Reincluded") && (currentType === None || currentType === Negative)
    - Scenario 2: status === "Absent" && latestDisputeOutcome === "Reject" && currentType === Positive
    - Scenario 3: status === "Absent" && latestDisputeOutcome !== "Reject" && currentType === Positive

    Pure function: no I/O, no async, no side effects. Takes typed inputs, returns Action[].
  </implementation>
</feature>
