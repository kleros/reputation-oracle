    ```graphql
    query GetItems($registryAddress: String!, $lastId: String!, $first: Int!) {
      items(
        where: { registryAddress: $registryAddress, id_gt: $lastId }
        orderBy: id
        orderDirection: asc
        first: $first
      ) {
        id
        itemID
        status
        metadata {
          key0
          key2
        }
        challenges(orderBy: challengeID, orderDirection: desc, first: 1) {
          disputeOutcome
          disputeID
          resolutionTime
        }
        stake
      }
    }
    ```

    - Create GraphQLClient with subgraphUrl
    - Cursor pagination loop:
      - lastId starts as "" (empty string)
      - pageSize = 1000
      - Loop: request ITEMS_QUERY with { registryAddress: registryAddress.toLowerCase(), lastId, first: pageSize }
      - Append items to allItems array
      - Update lastId = items[items.length - 1].id (the entity ID, NOT itemID — Pitfall 1)
      - Break if items.length === 0 or items.length < pageSize
    - Return allItems

    CRITICAL: Use `id` field (entity primary key) for cursor, NOT `itemID` (bytes32 hash). These are different fields. Pitfall 1 from RESEARCH.md.

    CRITICAL: Pass registryAddress.toLowerCase() to the where clause — subgraph addresses are lowercase.

    CRITICAL: Never use `skip` parameter. Only `id_gt` cursor. Per D-08.

    - Return the map

        - "revokeOnly": writeContract with args [action.agentId]

      - Per D-10: if receipt.status === "reverted", log error and throw (stop on first failure)

      - Log success: "TX {hash} confirmed for {action.type} agentId={action.agentId}"
