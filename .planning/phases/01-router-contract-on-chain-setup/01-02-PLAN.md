contract KlerosReputationRouter is Initializable, UUPSUpgradeable, OwnableUpgradeable {
    enum FeedbackType { None, Positive, Negative }

    int128 public constant NEGATIVE_VALUE = -95;

    function setKlerosAgentId(uint256 _klerosAgentId) external;

    error KRR_AlreadyHasPositiveFeedback(uint256 agentId);

}

    - `test_submitPositiveFeedback_emitsEvent`: Use `vm.expectEmit(true, true, false, true)` to check PositiveFeedbackSubmitted event with correct agentId and pgtcrItemId.

    - `test_submitNegativeFeedback_revokesPositiveThenSubmitsNegative`: First submit positive, then submit negative. Verify via getSummary: count=1, value=-95 (NOT average of +95/-95=0). Verify feedbackType == Negative.
    - `test_submitNegativeFeedback_fromNoneState`: Call submitNegativeFeedback when feedbackType is None (no prior feedback). Should succeed and set feedbackType to Negative. getSummary returns count=1, value=-95.

    **Scenario 3 -- Revoke only:**
    - `test_revokeOnly_removesPositiveFeedback`: Submit positive, then revokeOnly. Verify getSummary: count=0 (or count=1 with isRevoked). Verify feedbackType == None. Verify feedbackIndex reset to 0.

    - `test_reRegistration_afterNegative_allowsNewPositive`: Submit positive -> submit negative -> submit positive again. Verify getSummary returns positive value (history accumulates per project decision -- no revoke of old negative). feedbackType == Positive.

    **Authorization edge cases (ROUT-06):**
    - `test_revert_unauthorizedBot_submitPositive`: `vm.prank(unauthorized)`, expect revert `KRR_NotAuthorizedBot()`.
    - `test_revert_unauthorizedBot_submitNegative`: Same for submitNegativeFeedback.
    - `test_revert_unauthorizedBot_revokeOnly`: Same for revokeOnly.

    **State edge cases:**
    - `test_revert_doublePositive`: Submit positive twice, second call reverts with `KRR_AlreadyHasPositiveFeedback`.
    - `test_revert_doubleNegative`: Submit negative twice (via positive->negative->negative), second negative reverts with `KRR_AlreadyHasNegativeFeedback`.
    - `test_revert_revokeWithoutPositive`: Call revokeOnly when feedbackType is None, reverts with `KRR_NoPositiveFeedbackToRevoke`.
    - `test_revert_revokeWhenNegative`: Call revokeOnly when feedbackType is Negative, reverts with `KRR_NoPositiveFeedbackToRevoke`.

    **Owner management (ROUT-07):**
    - `test_setAuthorizedBot_onlyOwner`: Non-owner call reverts.
    - `test_setAuthorizedBot_emitsEvent`: Check BotAuthorizationChanged event.
    - `test_removeAuthorizedBot`: Set bot authorized, then set to false, then bot call reverts.

    Use `vm.expectRevert(abi.encodeWithSelector(KlerosReputationRouter.KRR_NotAuthorizedBot.selector))` pattern for custom error reverts.
